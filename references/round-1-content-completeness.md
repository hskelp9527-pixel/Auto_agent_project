# 第1轮：内容完整性检查 (content-completeness-check)

**权重**: ⭐ **60%** - 核心权重

## 目标

确保所有功能都有实际内容，不是空壳。

## 为什么这是第1轮？

- **功能完整性是核心**，权重占60%
- 内容缺失会让用户立即判断项目"未完成"
- 设计再好，没有内容也是零分
- 这是用户最先看到的东西

## 检查项目

### 1. 页面完整性检查（18分）

```bash
# 检查每个页面是否有实际内容
find app -name "page.tsx" -exec sh -c 'echo "=== $1 ===" && grep -E "占位|TODO|暂无|empty|placeholder" "$1"' _ {} \;
```

**必须避免的占位符**：
- ❌ 4字占位符（如"订单表单"、"库存列表"）
- ❌ "暂无数据"、"数据加载中"
- ❌ "TODO"、"待开发"、"coming soon"
- ❌ 空白组件（`<p>内容</p>`、`<div></div>`）
- ❌ 只有标题没有内容

### 2. 表单完整性检查（13分）

```bash
# 检查所有创建/编辑表单页面
find app -path "*/create/page.tsx" -o -path "*/edit/page.tsx"

# 检查表单是否包含必要元素
grep -r "input\|select\|textarea\|form" app/*/create/page.tsx
```

**表单最低要求**：
- ✅ 至少有3个以上输入字段
- ✅ 有提交/保存按钮
- ✅ 有必填字段标识
- ✅ 有表单验证
- ✅ 有取消/返回按钮

### 3. 示例数据检查（10分）

```bash
# 检查是否有足够的示例数据
grep -r "mockData\|EXAMPLE\|const.*=.*\[" lib/ data/

# 统计数据条数
grep -r "export const.*:" lib/ | grep -o "\[.*\]" | head -5
```

**最少数据要求**：
- ✅ 每个主要实体至少5-10条数据
- ✅ 数据字段完整（不是空对象）
- ✅ 数据有真实感（不是"测试1"、"测试2"）
- ✅ 关联数据之间有逻辑关系

### 4. 图表完整性检查（10分，如适用）

```bash
# 检查是否安装并使用了图表库
cat package.json | grep -E "recharts|chartjs|vis|echarts"
grep -r "from.*chart" app/ --include="*.tsx"
```

**图表最低要求**（如果有数据展示页面）：
- ✅ 至少有2个不同类型的图表
- ✅ 图表有真实数据源
- ✅ 图表有标题和图例
- ✅ 图表能正常渲染

### 5. CRUD功能检查（4分）

```bash
# 检查是否有增删改查功能
grep -r "create\|add\|edit\|update\|delete\|remove" app/ --include="*.tsx" | grep -i "button\|link"

# 检查store是否有对应方法
grep -r "add\|update\|delete" lib/store.ts
```

**必须具备**：
- ✅ 新建/添加按钮和页面
- ✅ 编辑功能（每个主要实体）
- ✅ 删除功能（每个主要实体）
- ✅ 查看详情功能
- ✅ 功能连接到数据store

### 6. 操作按钮检查

```bash
# 检查表格列表页是否有操作列
grep -r "操作\|Actions\|action" app/ --include="*.tsx" -A 2
```

**必须包含**：
- ✅ 编辑按钮
- ✅ 删除按钮
- ✅ 查看详情按钮
- ✅ 按钮有明确的图标或文字

### 7. 功能可用性检查 ⚠️ **关键项（5分）**

```bash
# 检查按钮是否有实际功能处理
# 1. 检查是否有onClick处理函数
grep -r "onClick.*handle\|onClick.*\(" app/ --include="*.tsx" | grep -E "delete|remove|edit"

# 2. 检查删除功能是否连接到store
grep -r "delete\|remove" app/ --include="*.tsx" | grep -v "className\|aria-label"

# 3. 检查详情页路由是否存在
find app -path "*/[id]/page.tsx"
```

**必须具备**：
- ✅ 删除按钮有onClick处理函数
- ✅ 删除功能调用store的delete方法
- ✅ 删除前有确认对话框（confirm）
- ✅ 查看按钮链接到实际存在的详情页路由
- ✅ 编辑按钮链接到编辑页或触发编辑功能
- ✅ 所有按钮都可以实际点击并执行操作

**常见错误**：
- ❌ 按钮存在但无onClick处理（点击无反应）
- ❌ 按钮有onClick但未连接到store（点击无效）
- ❌ 链接指向不存在的路由（404错误）
- ❌ 删除操作无确认对话框（误删风险）

### 8. 路由存在性检查 ⚠️ **零容忍项**

```bash
# 检查所有按钮链接的页面是否都存在
# 1. 提取所有Link和按钮的href/target
grep -r "href=\|to=" app/ --include="*.tsx" | grep -o "/[^\"]*" | sort -u

# 2. 检查编辑页是否存在
find app -path "*/[id]/edit/page.tsx"

# 3. 检查详情页是否存在
find app -path "*/[id]/page.tsx"

# 4. 检查创建页是否存在
find app -path "*/create/page.tsx"
```

**零容忍规则**：
- ✅ **所有按钮链接的页面都必须存在**
- ✅ **绝不出现404错误**
- ✅ **哪怕功能不齐全，页面也必须能打开**
- ✅ **每个主要实体至少有：列表页、详情页、编辑页**

**最低页面要求**（每个实体）：
- ✅ 列表页：显示数据列表
- ✅ 详情页：显示单条数据详情
- ✅ 编辑页：编辑数据
- ✅ 创建页：新建数据（核心实体）

**常见404错误模式**：
- ❌ `/[entity]/[id]/edit` - 编辑页不存在
- ❌ `/[entity]/[id]` - 详情页不存在
- ❌ `/[entity]/create` - 创建页不存在
- ❌ 链接href拼写错误

**评分标准**：
- **发现一个404，功能可用性5分全部扣除**
- **即使总分60分，出现404也视为不通过**

## 评分标准

**总分**：60分

| 项目 | 分数 |
|------|------|
| 页面完整性 | 18分 |
| 表单完整性 | 13分 |
| 示例数据 | 10分 |
| 图表完整性 | 10分 |
| CRUD功能 | 4分 |
| **功能可用性** | **5分** ⭐ |

**通过标准**：≥ 48分（80%）

## 不通过处理

如果不通过：
- 必须补充内容或修复功能
- 重新进入第1轮迭代
- 不能进入第2轮

## 特别注意

- 功能可用性是关键项（5分），如果按钮不可用或存在404，直接扣除全部5分
- 即使有60分，如果功能不可用或存在404，也视为不通过
- 零容忍：绝不出现404错误
