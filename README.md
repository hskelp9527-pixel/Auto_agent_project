# Self-Evolution System

基于 autoresearch 理念的开发项目自我迭代系统。

## 功能特性

- **第0轮：错误历史检查** ⚠️ **最高优先级** - 强制读取ERROR_LOG.md，避免重复已知错误
- **至少4轮迭代验证**：内容完整性（60%）、设计系统（20%）、代码质量（10%）、功能验证（10%）
- **源自 Andrej Karpathy 的 autoresearch 项目思想**，改造为通用开发项目 QA 领域
- **适用于任意开发类项目**（Web/移动/桌面/后端等），在 Web 应用等 UI 可见项目上表现最佳
- **确保零404、零按钮失效、零占位符**，真正做到"开箱即用"
- **根据项目实际情况可增加第5+轮专项验证**

## 核心规则

### ⚠️ 最高优先级规则

1. **ERROR_LOG.md强制使用**：
   - 第0轮必须读取ERROR_LOG.md
   - 每次迭代开始前都要检查ERROR_LOG.md
   - 发现未解决的错误：立即修复，不能继续
   - 发现类似问题：立即修复，不能继续
   - **禁止跳过第0轮直接进入第1轮**
   - **禁止只记录不读取ERROR_LOG.md**

### 其他核心规则

2. **持续验证至95分ACTUAL功能**：静态分析95分 ≠ 真实功能95分，必须浏览器验证到真实95分
3. **非必要不询问**：禁止在75分或其他中间分数询问用户，验证流程应自主完成
4. **CSS颜色对比度强制检查**：白色文字在白色背景上不可见，必须检查CSS配色
5. **UI可读性全面验证**：在有头浏览器中实际输入文本，确认所有内容可见

## ERROR_LOG.md 的重要性

**ERROR_LOG.md不是"死档案"，而是"活提醒"**：

- ✅ 记录所有遇到的错误
- ✅ **每次测试前强制读取**
- ✅ **检查是否有类似问题**
- ✅ **发现就解决**
- ❌ **禁止只记录不读取**

**使用流程**：
```
开始测试 → 读取ERROR_LOG.md → 检查类似问题 → 修复 → 继续测试
```

## 使用方法

在 Claude Code 中安装此 skill 后，当完成开发项目时自动触发自我迭代验证流程。

**验证流程**：
1. 第0轮：错误历史检查（ERROR_LOG.md）
2. 第1轮：内容完整性检查（60%）
3. 第2轮：设计系统质量检查（20%）
4. 第3轮：代码质量检查（10%）
5. 第4轮：浏览器功能验证（10%）
6. 根据需要增加第5+轮专项验证
